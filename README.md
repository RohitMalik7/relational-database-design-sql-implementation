# Relational Database Design and SQL Implementation

> **Author:** Rohit Malik | **Domain:** Database Engineering | **Type:** Academic Project | **Purpose:** Portfolio & Educational

---

## Overview

This project demonstrates end-to-end relational database design and implementation, covering both theoretical foundations and practical SQL development.

The work is structured into two stages -database analysis and database implementation -covering relational algebra, SQL querying, normalization, ERD design, schema implementation, constraints, and advanced SQL views.

---

## Project Objective

The objective of this project is to design a robust relational database system and implement it using SQL, ensuring data integrity, proper normalization, and efficient querying across both the conceptual and physical design stages.

---

## Assignment 1 – Database Analysis

This stage focuses on fundamental database concepts, query design, and normalization.

### Key Areas Covered

| Area | Description |
|---|---|
| **Relational Algebra** | Designed complex queries to extract meaningful insights from relational data |
| **SQL SELECT Queries** | Implemented queries using joins, aggregation, filtering, and subqueries |
| **Database Normalization** | Normalized schema up to Third Normal Form (3NF) to eliminate redundancy |
| **Anomaly Resolution** | Identified and resolved update, insertion, and deletion anomalies |
| **Conceptual ERD Design** | Developed an entity-relationship diagram to represent system structure |

---

## Assignment 2 – Database Design and Implementation

This stage focuses on building and implementing a full database system from schema to production-ready SQL.

### Key Areas Covered

| Area | Description |
|---|---|
| **ERD Refinement** | Refined conceptual model into a detailed logical and physical schema |
| **Data Dictionary** | Documented all entities, attributes, data types, and constraints |
| **SQL Table Implementation** | Created tables with primary keys, foreign keys, and constraints |
| **Data Integrity** | Applied CHECK, NOT NULL, and UNIQUE constraints throughout |
| **Referential Integrity** | Defined CASCADE and RESTRICT rules for relationship management |
| **Business Rules** | Enforced domain-specific validation rules through constraints |
| **SQL Views** | Created multiple views for data abstraction and real-world analysis |
